Building the DAX layer in Power BI
Turning L2's two CSV exports into measures a stakeholder can read at a glance
L2 produced two files: l2_exception_classified_dated.csv (99,441 order-level rows) and l2_exception_summary.csv (6-row severity rollup). On import, Power BI names each table after its source file — including the extension — so the working table name became l2_exception_classified_dated csv, with a space in it. Five measures were built on top of it.
Total Orders — baseline row count
Total Orders = COUNTROWS('l2_exception_classified_dated csv')
Counts every row in the table, exceptions and matches together. This is the denominator every other rate is measured against — expected to equal L0's 99,441.
Matched Orders — filtered count
Matched Orders = 
CALCULATE(
    COUNTROWS('l2_exception_classified_dated csv'),
    'l2_exception_classified_dated csv'[exception_type] = "OK"
)
CALCULATE re-evaluates COUNTROWS under a filter — here, only rows where exception_type = "OK". This is the count of orders that reconciled cleanly with no exception raised.
Match Rate % — the headline metric
Match Rate % = 
DIVIDE([Matched Orders], [Total Orders])
DIVIDE is used instead of / because it returns blank instead of an error if Total Orders is ever zero. Must be formatted as Percentage in the measure's Format field, or it displays as 0.98 instead of 98%.
Exception Orders — derived from the two measures above
Exception Orders = 
[Total Orders] - [Matched Orders]
Simple subtraction between two already-defined measures — DAX measures can reference each other, so this doesn't need its own CALCULATE.
Total Financial Exposure — conditional aggregation
Total Financial Exposure = 
SUMX(
    FILTER('l2_exception_classified_dated csv', 'l2_exception_classified_dated csv'[exception_type] <> "OK"),
    IF(
        'l2_exception_classified_dated csv'[exception_type] = "CANCELED_WITH_PAYMENT",
        'l2_exception_classified_dated csv'[payment_total],
        ABS('l2_exception_classified_dated csv'[diff])
    )
)
SUMX walks row-by-row through every exception (everything not "OK") and applies an IF to pick the right number per row. For most exception types, exposure is ABS(diff) — the reconciliation gap. CANCELED_WITH_PAYMENT is the one case where diff doesn't apply — there's no order total left to reconcile against — so exposure there is the full payment_total instead: money collected with no matching delivery.
High Severity Orders — reusing a measure inside CALCULATE
High Severity Orders = 
CALCULATE(
    [Exception Orders],
    'l2_exception_classified_dated csv'[exception_type] = "CANCELED_WITH_PAYMENT"
)
Same CALCULATE pattern as Matched Orders, but filtering to the single highest-severity exception type and reusing the Exception Orders measure instead of writing COUNTROWS again.
⚠ Table names with spaces need quotes
Every reference to the table above is wrapped in single quotes: 'l2_exception_classified_dated csv'. Typing the name bare — l2_exception_classified_dated csv[exception_type] — throws Failed to resolve name 'l2_exception_classified_dated', because DAX reads the space as the end of the identifier. Any table or column name containing a space, or starting with a number, needs single quotes around it. Using Power BI's autocomplete when typing formulas avoids this — it inserts the quoted name automatically.
